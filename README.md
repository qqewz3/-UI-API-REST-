# Пет-проект: Комплексное UI и API (REST) тестирование платформ Steam & Trello

Репозиторий содержит примеры тестовой документации (UI/UX) и автоматизированных тестовых коллекций (API) для демонстрации практических навыков QA-инженера. Проект моделирует реальные задачи коммерческого тестирования интернет-магазинов и таск-менеджеров.

## Используемые технологии и инструменты
* **Тест-дизайн:** Эквивалентное разделение, Граничные значения, Комбинаторика, Позитивное/Негативное тестирование.
* **Инструменты:** Postman, Google Docs / Sheets, DevTools
* **Автоматизация & API:** REST API, JavaScript (автотесты в Postman), Переменные коллекции, Сквозное связывание запросов (API Chaining).


## Часть 1: UI/UX Тестирование (Платформа Steam)

Разработана гибкая тестовая документация для критически важных модулей интернет-магазина (Регистрация, Поиск и Фильтрация, Корзина покупок, Локализация).

* **Чек-лист (Позитивный/Негативный):** Компактное верхнеуровневое покрытие основных сценариев использования.
* **E2E Тест-кейсы:** Детализированные пошаговые сценарии для проверки сложной бизнес-логики (например, ограничение на повторную покупку игры себе, если она уже есть в библиотеке).
* **Матрицы тест-дизайна:** Оптимизация более 20 проверок валидации полей ввода (Email, Пароль, Логин) в компактные таблицы, что сократило объем документации без потери тестового покрытия.

## Часть 2: Тестирование API и Автоматизация (Postman + JS)

В репозитории представлены две экспортированные коллекции Postman (`.json` файлы). Все запросы покрыты автоматическими JavaScript-проверками и успешно проходят коллекционные прогоны.

### 1. Коллекция: Steam Store API (`steam_postman_collection.json`)
* **Позитивные сценарии:** Извлечение массивов данных достижений (Dying Light 2) и списков купленных игр пользователя (`GetOwnedGames`).
* **Бизнес-логика:** Автотест на сравнение задекларированного счетчика `game_count` с реальной длиной массива данных.
* **Негативные тесты & Безопасность:** Обработка ошибок типов данных (`400 Bad Request`) и валидация прав доступа при поддельном токене (`403 Forbidden`) с парсингом HTML-ответа сервера через метод `.toLowerCase().include()`.

### 2. Коллекция: Trello API CRUD (`trello_postman_collection.json`)
Продвинутый интеграционный сценарий, демонстрирующий полный жизненный цикл объекта в базе данных (CRUD) за 2.6 секунды.
* **API Chaining:** Настроена динамическая передача переменных. Postman автоматически перехватывает ID созданной доски, передает его для создания колонки, затем перехватывает ID колонки для карточки (`POST`), модифицирует карточку (`PUT`) и полностью удаляет её в конце (`DELETE`), очищая за собой базу данных.

## Как запустить API тесты локально
1. Скачайте `.json` файлы коллекций из этого репозитория.
2. Откройте Postman и нажмите кнопку **Import** в левом верхнем углу.
3. Выберите скачанные файлы.
4. В настройках коллекции Trello во вкладке **Variables** укажите свои персональные `API_key` и `API_token`.
5. Нажмите на три точки у названия коллекции -> **Run collection** -> **Run**.
